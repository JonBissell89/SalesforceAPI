## Purpose

This integration pushes Lead records into **Salesforce production** via the Salesforce REST API. It supports three operations:

1. **Discover** — query the Lead schema so your integration adapts to custom fields automatically.
2. **Create** — insert a new Lead.
3. **Upsert** — insert-or-update a Lead by an External ID field, preventing duplicates and making retries safe.

## Prerequisites

Before these endpoints will work, a Lumina Salesforce administrator must complete must supply a client id and secret

<Warning>
  These endpoints are part of the standard Salesforce REST API. They will not work without a Connected App and an authorized integration user in the Lumina Solar org.
</Warning>

## High-level flow

```text
1. Authenticate   →  obtain an OAuth access token
2. Describe Lead  →  GET  /services/data/v60.0/sobjects/Lead/describe
3. Create Lead    →  POST /services/data/v60.0/sobjects/Lead
   — or —
   Upsert Lead    →  PATCH /services/data/v60.0/sobjects/Lead/{ExternalIdField}/{value}
```

## Base URL

All endpoints use the Lumina Solar Salesforce production instance:

```text
https://luminasolar.my.salesforce.com
```

<Tip>
  Use the **upsert** pattern whenever possible. It is idempotent — safe to retry without creating duplicates.
</Tip>

## Required fields

Every Lead submitted through this API must include: **FirstName**, **LastName**, **Email**, **Phone**, **Street**, **City**, **State**, **PostalCode**, **Lead_Campaign__c** (name of the partner company sending the lead), and **UTM_Source__c** (must be set to `Partner`).

See [Create a Lead](/docs/lead-api/create-lead) for the full required fields table and examples.
