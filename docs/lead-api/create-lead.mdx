---
title: "Create a Lead"
description: "Insert a new Lead record into Lumina Solar Salesforce production."
---

## Endpoint

```http
POST https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Lead
```

## Required fields

Every Lead submitted through this API **must** include the following fields:

| API Name | Type | Label | Notes |
|---|---|---|---|
| `FirstName` | string | First Name | |
| `LastName` | string | Last Name | |
| `Email` | email | Email | |
| `Phone` | phone | Phone | |
| `Street` | textarea | Street | |
| `City` | string | City | |
| `State` | string | State/Province | |
| `PostalCode` | string | Zip/Postal Code | |
| `Lead_Campaign__c` | picklist | Secondary Campaign Source | Name of the company sending the lead |
| `UTM_Source__c` | string | UTM Source | Must be set to `Partner` |

<Warning>
Requests missing any of these fields will be rejected. `Lead_Campaign__c` must contain the name of the partner company submitting the lead, and `UTM_Source__c` must always be set to `Partner`.
</Warning>

### External ID (recommended)

| API Name | Type | Label | Notes |
|---|---|---|---|
| `External_ID__c` | string | External ID | Required if you want a response body on upsert |

`External_ID__c` is not strictly required, but **you must include it if you want Salesforce to return a response** (record ID) on upsert. Without it, you cannot use the [upsert endpoint](/docs/lead-api/upsert-lead) and will not receive confirmation of which record was created or updated.

## Example request

```bash
curl -X POST "https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Lead" \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "FirstName": "Jon",
    "LastName": "Doe",
    "Email": "jon.doe@example.com",
    "Phone": "8025551234",
    "Street": "123 Main St",
    "City": "Saxtons River",
    "State": "VT",
    "PostalCode": "05154",
    "Lead_Campaign__c": "Acme Solar Partners",
    "UTM_Source__c": "Partner",
    "Company": "Doe Roofing LLC",
    "Average_Electric_Bill__c": 185.00
  }'
```

## Example response â€” success

```json
{
  "id": "00Q5e000001abc1EAA",
  "success": true,
  "errors": []
}
```

**HTTP 201 Created** is returned on success.

## Example response â€” validation error

```json
[
  {
    "message": "Required fields are missing: [LastName, Company]",
    "errorCode": "REQUIRED_FIELD_MISSING",
    "fields": ["LastName", "Company"]
  }
]
```

**HTTP 400 Bad Request** is returned when required fields are missing or values are invalid.

<Warning>
The create endpoint is **not idempotent**. Retrying a successful request creates a duplicate Lead.
Use [Upsert by External ID](/docs/lead-api/upsert-lead) instead for retry-safe ingestion.
</Warning>
