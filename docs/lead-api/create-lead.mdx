---
title: "Create a Sales Partner Lead"
description: "Insert a new Sales Partner Lead record into Lumina Solar Salesforce production."
---

## Endpoint

```http
POST https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Sales_Partner_Lead__c
```

## Required fields

Every Sales Partner Lead submitted through this API **must** include the following fields:

| API Name | Type | Label | Notes |
|---|---|---|---|
| `First_Name__c` | string | First Name | |
| `Last_Name__c` | string | Last Name | |
| `Email__c` | email | Email | |
| `Phone__c` | phone | Phone | |
| `Street__c` | textarea | Street | |
| `City__c` | string | City | |
| `State__c` | string | State | |
| `Postal_Code__c` | string | Postal Code | |
| `Lead_Campaign__c` | string | Lead Campaign | Name of the company sending the lead |
| `UTM_Source__c` | string | UTM Source | Must be set to `Partner` |
| `OwnerId` | reference | Owner | Your integration user's Salesforce ID (see [Authentication](/docs/lead-api/authentication#step-2--get-your-user-id-ownerid)) |

<Warning>
Requests missing any of these fields will be rejected. `Lead_Campaign__c` must contain the name of the partner company submitting the lead, `UTM_Source__c` must always be set to `Partner`, and `OwnerId` must be a valid Salesforce User or Queue ID retrieved from the `/chatter/users/me` endpoint.
</Warning>

### External ID (recommended)

| API Name | Type | Label | Notes |
|---|---|---|---|
| `External_ID__c` | string | External ID | Required if you want a response body on upsert |

`External_ID__c` is not strictly required, but **you must include it if you want Salesforce to return a response** (record ID) on upsert. Without it, you cannot use the [upsert endpoint](/docs/lead-api/upsert-lead) and will not receive confirmation of which record was created or updated.

## Optional fields

These fields are not required but can be included in the request body:

| API Name | Type | Label |
|---|---|---|
| `Company__c` | string | Company |
| `Mobile_Phone__c` | phone | Mobile Phone |
| `Average_Electric_Bill__c` | currency | Average Electric Bill |
| `Electric_Provider__c` | string | Electric Provider |
| `Appointment_Type__c` | string | Appointment Type |
| `Lead_Type__c` | string | Lead Sales Type |
| `County__c` | string | County |
| `Notes__c` | textarea | Notes |

## UTM and tracking fields

These fields capture marketing attribution data:

| API Name | Type | Label |
|---|---|---|
| `UTM_Source__c` | string | UTM Source |
| `UTM_Medium__c` | string | UTM Medium |
| `UTM_Campaign__c` | string | UTM Campaign |
| `UTM_Content__c` | string | UTM Content |
| `UTM_Term__c` | string | UTM Term |
| `Traffic_Source__c` | string | Traffic Source |
| `Landing_Page_URL__c` | textarea | Landing Page URL |
| `fbclid__c` | string | fbclid (Facebook Click ID) |
| `gclid__c` | string | gclid (Google Click ID) |
| `Channel__c` | string | Channel |

## Example request

```bash
curl -X POST "https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Sales_Partner_Lead__c" \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "First_Name__c": "Jon",
    "Last_Name__c": "Doe",
    "Email__c": "jon.doe@example.com",
    "Phone__c": "8025551234",
    "Street__c": "123 Main St",
    "City__c": "Saxtons River",
    "State__c": "VT",
    "Postal_Code__c": "05154",
    "Lead_Campaign__c": "Acme Solar Partners",
    "UTM_Source__c": "Partner",
    "OwnerId": "005XXXXXXXXXXXXXXX",
    "Company__c": "Doe Roofing LLC",
    "Average_Electric_Bill__c": 185.00
  }'
```

## Example response — success

```json
{
  "id": "a0ATV000001abc1EAA",
  "success": true,
  "errors": []
}
```

**HTTP 201 Created** is returned on success.

## Example response — validation error

```json
[
  {
    "message": "Required fields are missing: [Last_Name__c]",
    "errorCode": "REQUIRED_FIELD_MISSING",
    "fields": ["Last_Name__c"]
  }
]
```

**HTTP 400 Bad Request** is returned when required fields are missing or values are invalid.

<Warning>
The create endpoint is **not idempotent**. Retrying a successful request creates a duplicate record.
Use [Upsert by External ID](/docs/lead-api/upsert-lead) instead for retry-safe ingestion.
</Warning>
