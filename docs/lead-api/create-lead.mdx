---
title: "Create a Lead"
description: "Insert a new Lead record into Lumina Solar Salesforce production."
---

## Endpoint

```http
POST https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Lead
```

## Required fields

The following fields are **required** when creating a Lead (non-nullable and createable):

| API Name | Type | Label |
|---|---|---|
| `LastName` | string | Last Name |
| `Company` | string | Company |
| `Status` | picklist | Status |

<Note>
Several boolean fields also default as required (`DoNotCall`, `HasOptedOutOfEmail`, etc.) but Salesforce auto-defaults them to `false` if omitted. The three fields above are the ones you must always include in your payload.
</Note>

## Example request

```bash

  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "FirstName": "Jon",
    "LastName": "Doe",
    "Company": "Doe Roofing LLC",
    "Email": "jon.doe@example.com",
    "Phone": "8025551234",
    "City": "Saxtons River",
```

## Example response — success

```json
{
  "id": "00Q5e000001abc1EAA",
  "success": true,
  "errors": []
}
```

**HTTP 201 Created** is returned on success.

## Example response — validation error

```json
[
  {
    "message": "Required fields are missing: [LastName, Company]",
    "errorCode": "REQUIRED_FIELD_MISSING",
    "fields": ["LastName", "Company"]
  }
]
```

**HTTP 400 Bad Request** is returned when required fields are missing or values are invalid.

<Warning>
The create endpoint is **not idempotent**. Retrying a successful request creates a duplicate Lead.
Use [Upsert by External ID](/docs/lead-api/upsert-lead) instead for retry-safe ingestion.
</Warning>
