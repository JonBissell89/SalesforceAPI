## Endpoint

```http
GET https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Sales_Partner_Lead__c/describe
```

## Purpose

The Describe endpoint returns the full schema for the Sales Partner Lead object, including every field's API name, data type, length, whether it is required, and valid picklist values. Use it to build integrations that adapt automatically when custom fields are added.

## Example request

```bash
curl -X GET "https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Sales_Partner_Lead__c/describe" \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

## How to use the response

| Property | Use |
|---|---|
| `createable` | If `true`, the field can be set when creating a record. |
| `updateable` | If `true`, the field can be modified on an existing record. |
| `nillable` | If `false`, the field is required (must have a value). |
| `type` / `length` | Validate your payload values before sending. |
| `picklistValues` | Only submit `active: true` values for picklist fields. |

<Tip>
Filter fields by `createable: true` to build a dynamic payload.
Check `nillable: false` to identify required fields.
</Tip>
