## Endpoint

```http
GET https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Lead/describe
```

## Purpose

The Describe endpoint returns the full schema for the Lead object, including every field's API name, data type, length, whether it is required, and valid picklist values. Use it to build integrations that adapt automatically when custom fields are added.

## Example request

```bash
curl -X GET "https://luminasolar.my.salesforce.com/services/data/v60.0/sobjects/Lead/describe" \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

```json
{
  "name": "Lead",

    {
      "name": "LastName",

      "picklistValues": []
    },
    {
      "name": "Company",

      "length": 255,
      "nillable": false,

    },
    {
      "name": "LeadSource",

      "createable": true,

      "picklistValues": [
        { "value": "Web", "active": true },
        { "value": "Phone Inquiry", "active": true }
      ]
    }
  ]

```

## How to use the response

| Property | Use |
|---|---|
| `createable` | If `true`, the field can be set when creating a Lead. |
| `updateable` | If `true`, the field can be modified on an existing Lead. |
| `nillable` | If `false`, the field is required (must have a value). |
| `type` / `length` | Validate your payload values before sending. |
| `picklistValues` | Only submit `active: true` values for picklist fields. |

<Tip>
Filter fields by `createable: true` to build a dynamic payload.
Check `nillable: false` to identify required fields.
</Tip>
